# TravelPlacesTracker

Веб-додаток "Трекер місць для подорожей":
Завдання:
Створити веб-додаток, де користувачі можуть створювати список місць, які вони хочуть відвідати на основі знайдених згідно пошуку за параметрами (ключеве слово, геолокація).
Опис:
Використати будь-який публічний API (наприклад  Foursquare Places API), для отримання інформації про популярні туристичні місця.
Користувач може додавати місця у свій список бажань (wishlist) і переглядати деталі про кожне місце: рейтинг, тип, фотографії, поради\відгуки.
Реалізувати простий кеш, щоб не робити повторні запити до API для того самого міста протягом 10 хвилин.
Список збережених місць повинен зберігатися у локальному сховищі, щоб не зникали при перезавантаженні сторінки.

# Start

Для запуску введіть **ng serve --proxy-config proxy.conf.json** в консолі.
Я використовую proxy.conf.json file, бо отримую CORS помилки в браузері через використання API з іншого домену.

Оскільки нема логування для тестового проекту, я зберігаю ключ API та посилання на Foursquare API у environments.ts. Це не є безпечно, але я це зробив виключно для перевірки і тестування цього завдання.

# Опис реалізації

Користувач може ввести ключове слово, місто або використати свою поточну геолокацію. Також можна вказати радіус пошуку та кількість результатів.

Після пошуку додаток показує список знайдених місць у вигляді карток. На картці відображається назва, категорія, місто, адреса, рейтинг, відстань та короткий опис, якщо ці дані повертає API.

Користувач може додати місце у wishlist або видалити його звідти. Wishlist зберігається у localStorage, тому список не зникає після перезавантаження сторінки.

Для кожного місця можна відкрити детальну інформацію. У деталях показуються категорія, адреса, години роботи, статус відкрито/закрито, рейтинг, телефон, сайт, фото та опис, якщо ці дані повертає API.

Я також реалізував простий кеш для результатів пошуку та деталей місця. Дані зберігаються у localStorage на 10 хвилин, щоб не робити повторні запити до API для однакових параметрів.

До інтерактивних об'єктів добавив анімацію відкривання/закривання для покращення UX.
